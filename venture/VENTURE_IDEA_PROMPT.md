# **Poker Tool Product Exploration Prompt**

Context  
I have built a personal poker tool that logs interesting hands during play and performs session analysis. The tool is currently a local HTML/JS application and includes features such as:

* live hand logging  
* session review  
* interesting hand ranking  
* board texture analysis  
* archetype insights  
* behavioral summaries

The tool is optimized for quickly capturing key decisions during play.

Idea  
Explore turning this into a commercial product aimed primarily at **live poker players**.

Possible positioning:

“AI-powered live poker session tracker and trainer.”

Target users:

* recreational live players  
* aspiring serious players  
* players without access to traditional hand history trackers

Potential features:

* fast mobile hand logging  
* AI hand review and feedback  
* mistake detection  
* trainer drills generated from real hands  
* bankroll tracking  
* game discovery (optional)

Constraints:

* 40–50 hours available for an MVP sprint  
* prefer rapid AI-assisted development  
* focus on solving one real problem extremely well

Goals for discussion:

1. Identify the **smallest sellable product**  
2. Define the **one killer feature**  
3. Map a **1-week development roadmap**  
4. Evaluate realistic monetization

The conversation should combine:

* product strategy  
* startup thinking  
* pragmatic engineering decisions  
* AI-assisted development workflows.
